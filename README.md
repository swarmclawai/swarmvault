# SwarmVault

SwarmVault is a local-first LLM knowledge base compiler.

It turns raw research material into a persistent markdown wiki, a structured graph, and a local search index so your work compounds over time instead of disappearing into chat history. The core loop is simple:

1. Ingest source material into `raw/`
2. Compile those sources into `wiki/` and `state/graph.json`
3. Query the vault and save useful answers back into the wiki
4. Lint the vault for stale pages, missing citations, and drift

## What It Does

SwarmVault is built for the workflow Andrej Karpathy described around LLM-maintained personal knowledge bases: raw sources go in, the wiki is compiled by the model, and future questions operate on durable artifacts instead of one-off prompts.

The open source runtime gives you:

- Markdown-first outputs that stay usable in Obsidian or plain Git
- A structured graph artifact for relationships, provenance, and freshness
- Local search over the compiled vault
- CLI workflows for ingest, compile, query, lint, and graph serving
- Pluggable LLM providers, including OpenAI, Ollama, Anthropic, Gemini, generic OpenAI-compatible APIs, and custom adapters

## Why This Exists

Most "chat with your docs" tools answer a question and then forget the work. SwarmVault treats the vault itself as the product. Summaries, derived pages, saved answers, and graph structure are all persistent artifacts that can be reviewed, versioned, and reused.

The goal is not another ephemeral RAG UI. The goal is a compounding research workspace.

## Install

SwarmVault requires Node `>=24`.

```bash
npm install -g @swarmvaultai/cli
```

This installs the `swarmvault` command. The legacy alias `vault` is also available.

## Quickstart

```bash
mkdir my-vault
cd my-vault
swarmvault init
swarmvault ingest https://example.com/article
swarmvault compile
swarmvault query "What are the main concepts?" --save
swarmvault lint
swarmvault graph serve
```

After the first compile, your workspace will contain:

- `raw/`: immutable source files and captured assets
- `wiki/`: generated markdown pages and saved outputs
- `state/`: manifests, extracts, graph data, and search index
- `agent/`: agent-specific rules generated by `swarmvault install`

## Core Workflow

### `ingest`

Adds a local file path or URL to the vault and records an immutable manifest in `state/manifests/`.

### `compile`

Reads the manifests, extracts structure and claims, writes wiki pages, updates `state/graph.json`, and refreshes the local search index.

### `query`

Asks the compiled vault a question. With `--save`, the answer is written back into `wiki/outputs/` so it becomes part of the vault instead of staying transient.

### `lint`

Runs vault health checks, including stale-page detection and other anti-drift signals.

### `graph serve`

Serves the local graph explorer so you can inspect sources, concepts, and entities as separate layers.

## Providers

SwarmVault is provider-driven, not provider-locked.

Built-in provider types:

- `openai`
- `ollama`
- `anthropic`
- `gemini`
- `openai-compatible`
- `custom`
- `heuristic` for local smoke tests and offline defaults

Inside an existing `swarmvault.config.json`, the `providers` and `tasks` sections can look like this:

```json
{
  "providers": {
    "primary": {
      "type": "openai-compatible",
      "baseUrl": "https://your-provider.example/v1",
      "apiKeyEnv": "OPENAI_API_KEY",
      "model": "gpt-4.1-mini",
      "apiStyle": "chat",
      "capabilities": ["chat", "structured"]
    }
  },
  "tasks": {
    "compileProvider": "primary",
    "queryProvider": "primary",
    "lintProvider": "primary",
    "visionProvider": "primary"
  }
}
```

Compatibility is capability-driven. SwarmVault does not assume every OpenAI-compatible endpoint supports the full OpenAI surface.

## Packages

- `@swarmvaultai/cli`: the globally installable `swarmvault` command
- `@swarmvaultai/engine`: the core ingest/compile/query/lint/runtime library
- `@swarmvaultai/viewer`: the graph viewer package used by `swarmvault graph serve`

## Current Status

This is an early open source release. The core loop works, but a few edges are still intentionally simple:

- The default local provider is heuristic and meant for smoke tests, not production-quality synthesis
- The graph viewer is functional but still lightweight
- Node 24 currently emits an experimental warning for the built-in `node:sqlite` module used by the local search layer
