# Worked Examples

These small example vault inputs mirror the trust and capture workflows used in docs and smoke tests.

- `code-repo/` shows repo-aware code ingest plus compile/benchmark flow
- `mixed-corpus/` shows markdown, remote assets, and graph/report flow
- `capture/` shows opinionated `swarmvault add` capture inputs
- `book-reading/` shows chapter-by-chapter fan wiki construction with compounding character and theme pages
- `research-deep-dive/` shows incremental research synthesis with contradiction detection
- `personal-knowledge-base/` shows a personal Memex vault with journals, health logs, podcast notes, and goals
- `large-repo/` shows the B.7 large-repo graph knobs plus the B.8 per-source-class benchmark breakdown against a mock repo that hits every source class

They are intentionally small. Their job is to make the expected workflow obvious and to serve as stable example material for docs and release validation.
